# 便便打工人 | Poo-to-Pay

一个轻松、可爱、可离线使用的排便生活记录工具。它把每天的排便打卡转换成一笔“工资”，用一点游戏化反馈，帮助用户更自然地关注饮水、作息和身体状态。

## 核心功能

- 每日打卡：记录时间、排便状态、体感和备注
- 工资结算：按单次金额累计今日工资和历史总收入
- 工资流水：一键结算待发工资，结算后清零并保留结算时间、金额和包含的打卡数
- 记录管理：可删除待结算的错误打卡，已结算记录会锁定以保证工资流水准确
- 今日看板：查看目标进度、连续开工、本周工资和记录
- 饮水追踪：快速增加 250 ml / 500 ml 饮水量
- 趋势统计：通过近 7 天柱状图观察排便节奏
- 历史记录：搜索、回顾和导出 CSV
- 健康建议：根据饮水和近期状态提供生活建议
- 个人设置：修改称呼、工资和每日目标
- 本地存储：无需账号、后端或第三方服务

## 项目介绍

Poo-to-Pay 面向希望养成规律生活习惯、但不喜欢严肃健康打卡的人。产品用“每次上班都有回报”的轻量叙事，把容易被忽略的身体信号变成可见的进度和正反馈，同时保留温和、不过度焦虑的健康提示。

当前版本是纯 HTML、CSS 和 JavaScript 单页应用，无构建步骤、无运行时依赖，适合直接打开或部署到静态托管服务。

## 运行

直接双击 `index.html` 即可打开。也可以在项目目录运行：

```powershell
npx serve .
```

## 分享给朋友

这个版本是纯静态页面，数据保存在每个浏览器自己的 `localStorage` 中。因此把页面放到公网后，朋友可以访问和使用自己的记录，但不会自动看到你的记录；这也是当前版本无需账号、无需后端的原因。

临时给朋友试用，可以在项目目录运行：

```powershell
npx serve . --listen 0.0.0.0:4173
cloudflared tunnel --url http://localhost:4173
```

第二条命令会生成一个临时 `https://*.trycloudflare.com` 地址，适合短时间体验。电脑关机或命令停止后，地址就不可用了。

想使用自己的 `ai-chef.cn`，建议把这个目录部署到 Cloudflare Pages、Vercel 或 Netlify，再在服务商后台绑定域名。静态托管通常不需要购买服务器；域名 DNS 按服务商提供的记录配置即可。

如果你们两个人要共享同一份工资和打卡数据，当前 `localStorage` 方案不够，需要增加登录和云端数据库。比较合适的下一步是接入 Supabase，建立用户、打卡记录和结算流水三张表，并按用户权限隔离数据；在此之前不要把浏览器本地存储当作多人共享数据源。

## 后续值得增加

- JSON 备份与恢复，避免清浏览器数据后丢失记录
- 多用户登录和权限，支持你与朋友各自使用或共享一个小组
- 自定义结算周期，例如按周或按月结算
- 编辑已结算记录，或提供可审计的撤销结算流程
